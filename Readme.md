# ThreadPool on C++

Небольшой pet-project с реализацией собственного пула потоков на C++.

Проект показывает, как можно организовать:
- очередь задач,
- пул рабочих потоков,
- ожидание задач через `std::condition_variable`,
- получение результата через `std::future`,
- упаковку задач через `std::packaged_task`.

Это учебная реализация, которая помогает разобраться в базовых механизмах многопоточности в современном C++.

---

## Что умеет

- создаёт фиксированное количество рабочих потоков;
- принимает задачи в очередь;
- выполняет задачи параллельно;
- поддерживает задачи без возвращаемого значения;
- поддерживает задачи с возвращаемым значением через `std::future`;
- позволяет передавать функции, лямбды и аргументы к ним.

## Идея реализации

### Основные поля класса `ThreadPool`

- `tasks_` — очередь задач;
- `threads_` — вектор рабочих потоков;
- `cv_` — условная переменная для пробуждения потоков;
- `m_` — мьютекс для защиты очереди;
- `flag_stop_` — флаг завершения работы пула.

### Как это работает

1. При создании `ThreadPool` запускается заданное количество потоков.
2. Каждый поток выполняет метод `worker()`.
3. Если задач нет, поток засыпает на `cv_.wait(...)`.
4. Когда новая задача добавляется через `submit(...)`, один поток пробуждается.
5. Поток забирает задачу из очереди и выполняет её.
6. Если пул останавливается и задач больше нет, поток завершает работу.

---

## Метод `submit`

Метод `submit` принимает вызываемый объект и его аргументы, затем:

1. определяет тип результата через `std::invoke_result_t`;
2. упаковывает задачу в `std::packaged_task`;
3. получает `std::future`;
4. кладёт задачу в очередь;
5. уведомляет один рабочий поток;
6. возвращает `future` вызывающему коду.

Это позволяет получить результат асинхронного вычисления позже через `.get()`.

---

## Пример использования

В `main()` создаётся пул из 4 потоков:

```cpp
ThreadPool pool(4);
Далее в пул отправляются 10 задач:
for(int i = 0; i < 10; ++i) {
    pool.submit([i]{
        std::osyncstream(std::cout) << "task " << i << "\n";
    });
}

Потом отправляются ещё две задачи с возвращаемым значением:
auto f = pool.submit([]{
    return 42;
});

auto f2 = pool.submit(sum, 1, 3);

Результаты получаются через future:
std::cout << f.get() << "\n";
std::cout << f2.get() << "\n";
```
